# Geochemical Solver Plan

## Purpose

This document defines an initial plan for a geochemical solver module that complements the existing atmospheric chemistry and protocell/environment work. The geochemical solver will handle rock-water and hydrothermal aqueous chemistry using PHREEQC-style modeling, and couple geological outputs into surface and protocell environments.

## Goals

- Model internal geological heat transport and temperature structure
- Simulate rock-water geochemical equilibrium and kinetic reactions
- Produce aqueous and mineral composition outputs for surface/vent delivery
- Provide boundary fluxes that can feed the atmosphere, ocean, and protocell environment
- Keep geochemical solver modular and compatible with existing environment and atmosphere layers

## High-level architecture

The geochemical solver will have three main components:

1. Thermal transport / geothermal state
2. PHREEQC-based aqueous geochemistry
3. Delivery and coupling to surface/volcanic environments

### 1. Thermal transport

This component computes a steady or time-dependent temperature distribution in the subsurface/rock column.

Key functions:
- `compute_geothermal_gradient()`
- `solve_thermal_diffusion()`
- `compute_internal_heat_source()`

Inputs:
- internal heat production (radiogenic / tidal / magma)
- rock thermal properties
- initial thermal profile or boundary temperatures
- fluid flow / advective transport if applicable

Deliverables:
- temperature profile as a function of depth/position
- heat flux at boundary layers

### 2. PHREEQC-based aqueous geochemistry

This component will use `phreeqpy` and the PHREEQC database to calculate aqueous speciation, mineral saturation, and kinetic reactions.

Key functions:
- `initialize_phreeqc_model()`
- `build_solution_string()`
- `run_phreeqc()`
- `extract_aqueous_composition()`
- `extract_mineral_fluxes()`

Inputs:
- water composition / precursor solutes
- temperature and pressure from thermal model
- mineral assemblage / rock composition
- redox state, pH, and gas fluxes

Deliverables:
- aqueous species concentrations
- mineral dissolution/precipitation rates
- organics or reduced species generated by hydrothermal/geochemical reactions

### 3. Delivery and coupling

This component maps geochemical outputs to surface/external reservoirs and to existing environment models.

Key functions:
- `compute_surface_fluxes()`
- `apply_geochemical_outputs_to_environment()`
- `update_external_field()`

Coupling targets:
- `Environment.external_field` for protocell supply
- volcanic/hydrothermal flux boundary conditions
- atmospheric/ocean boundary species input

## Practical workflow

1. Compute geothermal state for the rock/vent system
   - use a 1D or layered thermal model
   - solve for temperature vs depth until equilibrium or transient state is reached

2. Build PHREEQC inputs at relevant locations
   - surface fluid composition
   - subsurface/hydrothermal fluid composition
   - rock/mineral assemblage

3. Run PHREEQC for equilibrium/kinetic geochemistry
   - determine dissolved species, pH, redox, mineral phases
   - optionally compute time-dependent reaction progress using kinetics

4. Translate PHREEQC output into environment fluxes
   - volcanic gas and fluid discharge
   - dissolved organics and reduced species
   - mineral precipitation/dissolution mass balance

5. Deliver outputs into the atmospheric and protocell simulation layers
   - add species to `Environment.external_field`
   - use fluxes as boundary conditions for surface/hydrothermal environments
   - feed proto-metabolite source terms for protocells

## Recommended first implementation

- Create `src/geochemical_solver/` package
- Add a lightweight PHREEQC wrapper module, e.g. `phreeqc_model.py`
- Add a thermal model module, e.g. `thermal_model.py`
- Add a coupling module, e.g. `coupling.py`
- Add a plan and interface definitions in `PLAN.md`

## Interface considerations

The geochemical solver should expose:

- `GeochemicalSolverState`
  - temperature_profile
  - pressure_profile
  - aqueous_composition
  - mineral_state
  - boundary_fluxes

- `run_geochemical_step(geo_state, env_state, dt)`
  - uses thermal and geochemical models to update the geochemical state
  - returns fluxes for coupling to `Environment`

- `build_phreeqc_input(env_state, geo_state)`
  - translates current environment and rock state into PHREEQC input

## Integration with existing codebase

- Use `src/environment/volcanic_rocks/volcanic_rock_env.py` and/or `src/environment/hydro_env/hydrothermal_env.py` as coupling points
- Keep atmospheric chemistry in `src/exo_chem/easy_chem_driver.py`
- Use geochemical outputs to update:
  - `Environment.external_field`
  - protocell reaction inputs
  - local fluid composition in hydrothermal environments

## Next milestones

1. Add `src/geochemical_solver/__init__.py`
2. Add `src/geochemical_solver/phreeqc_model.py` with a basic `IPhreeqc` wrapper
3. Add `src/geochemical_solver/thermal_model.py` with a simple 1D thermal diffusion solver
4. Add `src/geochemical_solver/coupling.py` to create environment fluxes
5. Add an example driver or notebook showing PHREEQC output for a simple hydrothermal fluid

## Notes

- PHREEQC is the preferred solver for aqueous rock-water geochemistry; do not reuse `EasyChem` for this domain.
- The thermal model should be simple initially: one-dimensional steady state with an internal heat source, then evolve to more complex transient/advection if needed.
- The geochemical solver should remain modular so the atmosphere and protocell code can use it as a separate backend.
